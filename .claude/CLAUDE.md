# Japanese Technical Writing Rules

日本語のテキストを出力する際は、以下のルールに従うこと。
これらは `@textlint-ja/textlint-rule-preset-ai-writing` に基づくルールである。

---

## 1. リストのAI的フォーマットを使わない (no-ai-list-formatting)

- `- **太字**: 説明文` のような機械的なリスト書式を使わないこと
- リスト項目に装飾的な絵文字を使わないこと
  - 禁止絵文字: ✅ ❌ ⭐ ✨ 💯 ⚠️ ❗ ❓ 💥 🔥 ⚡ 💪 🚀 💡 🤔 💭 🧠 🎯 📈 📊 🏆 👍 👎 😊 😎 🎉 🌟 📝 📋 ✏️ 🖊️ 💼

## 2. 誇大表現を使わない (no-ai-hype-expressions)

以下のような誇張・煽情的な表現を避けること。

### 絶対性・完全性の表現
- 革命的な / ゲームチェンジャー / 世界初の / 究極の / 完全に / 完璧な / 最高の / 最先端の / 大幅に

### 抽象的・煽情的な効果の表現
- 魔法のように / 奇跡的な / 驚異的な / 可能性を解き放つ / 潜在能力を引き出す / 民主化する / スーパーチャージ / 驚嘆させ

### 予測的・権威的な表現
- 業界を再定義 / 未来を変える / パラダイムシフト / 不可避の / 新たな基準を設定 / 次世代の / フロンティアを開拓 / 根本的に変革

## 3. AI的な強調パターンを使わない (no-ai-emphasis-patterns)

- 絵文字と太字を組み合わせた強調（例: 💡 **ポイント**）を使わないこと
- 以下の語句を太字にしないこと: 注意, 重要, ポイント, メモ, 参考, 補足, 確認, チェック, 推奨, おすすめ, 検出される例, 推奨される表現, 良い例, 悪い例, 例, サンプル, 使用例, 設定例
- 見出し（#）の中で太字（**）を使わないこと（見出し自体が強調なので冗長）

## 4. コロン継続パターンを使わない (no-ai-colon-continuation)

- 述語（動詞・形容詞・助動詞）の後にコロン（:/：）を付けて、その後にコードブロック・リスト・引用・表を続ける英語的なパターンを避けること
- 名詞の後のコロンは許容される

### 悪い例
```
以下のコマンドを実行します:
```

### 良い例
```
以下のコマンドを実行します。
```
```
実行コマンド:
```

## 5. テクニカルライティングガイドライン (ai-tech-writing-guideline)

### 冗長表現の排除
| 避ける表現 | 推奨する表現 |
|---|---|
| まず最初に | まず |
| あらかじめ予測 | 予測 |
| することができます | できます / します |
| する必要があります | してください / します |
| 言うまでもなく | （削除） |

### 能動態の使用
- 受動態（「が行われ（て/る/ます）」「によって実行され」等）を避け、能動態で書くこと
- 名詞化表現（「の変更を行う」「の実装を実施」等）を避け、直接的な動詞を使うこと

### 具体性の確保
以下のような曖昧な表現を避け、具体的な数値や説明に置き換えること:
- 「高速なパフォーマンス/処理/動作」
- 「大幅に向上/改善/削減」
- 「効率的な」
- 「適切な」
- 「必要に応じて」

### 用語の一貫性
同一ドキュメント内で以下のような同義語を混在させないこと:
- ユーザー / クライアント / 顧客
- 設定画面 / 設定ページ / 環境設定
- です・ます調 と だ・である調 の混在

### 文書構造
- 接続表現 + 句点で終わる段落の直後に箇条書きを始める機械的パターンを避けること
- 「第一に/まず → 第二に/次に → 第三に/最後に」のような定型的な順序パターンを避けること
